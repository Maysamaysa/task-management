# Task Management API

A robust, enterprise-grade REST API for managing tasks and users, built with [NestJS](https://nestjs.com/) and [PostgreSQL](https://www.postgresql.org/).

## Features

- **Authentication & Authorization**: Secure JWT-based login featuring short-lived access tokens and long-lived refresh tokens.
- **RBAC (Role-Based Access Control)**: Granular access control distinguishing between `admin` users (full system access) and `employee` users (restricted to their own tasks).
- **Task Management**: Full CRUD capabilities for task tracking including statuses (`TODO`, `IN_PROGRESS`, `DONE`).
- **Data Integrity**: Implements `ParseUUIDPipe` validation for reliable routing and Postgres `ON DELETE CASCADE` constraints to prevent orphaned database records.
- **Robust Testing suite**: Achieving ~100% statement and branch test coverage using `Jest`.

## Technologies Used

- **Framework**: NestJS v11
- **Database**: PostgreSQL 
- **ORM**: Sequelize (with `sequelize-typescript`)
- **Authentication**: Passport & JWT (`jsonwebtoken`, `bcrypt`)
- **Documentation**: Swagger UI (`@nestjs/swagger`)
- **Testing**: Jest with `ts-jest` transpilation
- **Containerization**: Docker & Docker Compose

## Prerequisites

- Node.js (v22+)
- Docker and Docker Compose (For running the database locally)

## Installation

1. Install project dependencies:
```bash
npm install
```

2. Create a `.env` file in the root directory and populate it with your environment variables:
```env
# Server
PORT=3000

# Database Configuration
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=your_username
DB_PASSWORD=your_password
DB_DATABASE=task_management

# JWT Secrets
JWT_SECRET=super_secret_key_change_in_prod
JWT_REFRESH_SECRET=super_refresh_secret_key_change_in_prod
```

3. Spin up the PostgreSQL database via Docker Compose:
```bash
docker compose up -d postgres_db
```

or start them all in 
```bash
docker compose up -d
```

## Running the Application

```bash
# development
npm run start

# watch mode (recommended for development)
npm run start:dev

# production mode
npm run start:prod
```

The API will be accessible at `http://localhost:3000`.

## Swagger API Documentation

Once the application is running, you can explore the endpoints interactively via the autogenerated Swagger UI.

Navigate to: [http://localhost:3000/api/docs](http://localhost:3000/api/docs)

### Core Endpoints Overview

#### Auth
- `POST /auth/register` - Register a new employee account.
- `POST /auth/login` - Authenticate and receive an Access Token & Refresh Token.
- `POST /auth/refresh` - Refresh an expired Access Token.
- `GET /auth/profile` - Verify current logged-in identity.

#### Users (Admin Only)
- `GET /users` - Retrieve all registered users.
- `GET /users/:id` - Retrieve a specific user by UUID.
- `PATCH /users/:id` - Update user details (name, role, password).
- `DELETE /users/:id` - Delete a user structure (cascades to their tasks).

#### Tasks (RBAC Enforced)
- `POST /tasks` - Create a new task.
- `GET /tasks` - List tasks (Employees only see theirs, Admins see all). Includes optional `?status=` query filter.
- `GET /tasks/:id` - View a specific task.
- `PATCH /tasks/:id` - Update task details or change status (`TODO`, `IN_PROGRESS`, `DONE`).
- `DELETE /tasks/:id` - Delete a specific task.
